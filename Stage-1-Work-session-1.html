Basics of the World Wide Web

Some "Web jargon" simplified:

HTML documents: This is the main type of document on the Web. They are what holds the Web together!
URL's: Think of these as the "addresses" or "names" of your documents. You use a URL to refer to a document, like you would
use a friend's name, to refer to them!
HTTP: This is the protocol with which different "parts" of the Web communicate.
Web applications: These are what really matter in the end. Think of them as the paint that makes a painting, on a white canvas.

What exactly is the Web?

The Web is mainly composed of a bunch of HTML documents. (HTML stands for HyperText Markup Language). These HTML documents
are "linked" together, and the "links" that connect them are called hyperlinks. This "linking" is what makes the Web "web-like"!
You can link a bunch of other documents and files to the HTML documents, but the HTML documents are the main building blocks of the Web.

The major components of the Web

You: You type something into your..
Computer and browser: Your computer is running the web browser that you type into. (Examples of browsers are Chrome, Firefox, Safari, you
know). So, your computer sends your requests (using the protocol called HTTP that we mentioned earlier) via the...
Internet: The requests are sent via the Internet to...
Servers: The servers respond to your requests by sending back files, via the internet, to your computer and browser, delivering them to you!
("Servers" are basically some other computers sitting somewhere else in a closet, hosting files, as opposed to your computer that you use to browse
files. Don't let the big words get to you.)

HTML - up close and personal!

HTML is the core of the Web, but what's it made of? Well, it's made up of:
Text content: This is the writing you see.
Mark up: This is how this writing is displayed.
References to other documents: Like references to videos or pictures.
Links to other pages: Which are..well...links to other pages. Pretty obvious stuff.

Mark up

HTML mark up is made up of things called tags. A tag is written in between a "lesser than" symbol and a "greater than" symbol. Here's an example:
<tag> content </tag>
The first tag is called the opening tag, and the last tag is the closing tag. The name of the tag is the same both for opening and closing, but the name
in the closing tag has a slash "/" typed before it. The word "content" in the example refers to what you are trying to use mark up for. The content can be
text, or it can even be other tags! The entire thing, meaning the opening tag, content, and the closing tag, is referred to as "the element".

Different types of tags:

The <b> tag:"b" stands for "bold", and that's what it does to contents of the element! So, to make the word "contents" appear in bold, I typed
something like this: <b>contents</b> . That made it become bold!
The <em> tag:"em" stands for "emphasis". This tag makes content italic. So, guess what happens if you miss the closing end tag? The mark up
that the tag performs is implemented for everything that comes after it. What do we learn from this? Be careful! Your computer only implements
what you tell it to do!

HTML attributes:

Let's represent this as follows:
<tag ATTR = "value"> content </tag>
In this example, we have the opening and closing tags as we've previously mentioned, but the tag has an attribute, which in this case has been given
the name ATTR. Tags can have multiple attributes, and as you can see, attributes have values.
An example of a tag that has an atttrubute is:
The <a> tag:"a" stands for "anchor". This kind of tag is used to make links. An anchor tag looks something like this:
<a href = "https://www.udacity.com">education</a>
In the above example, href is the attribute, its value is a URL which in this case is www.udacity.com , and the content is the word "education".
If this anchor tag was rendered into HTML, we would see the word "education" having become a link, to the URL of the udacity website.
This is what it would become: education . Go ahead and click it!
The <img> tag: "img" stands for image. This is the tag you use to make a link to an image, and it looks something like this:
<img src ="url" alt="text">
In this case, the attribute is src, which stands for source, and the value of this attribute is a URL to the image you want to show.
The other attribute in this element is the alt tag, wich stands for alternate. The value of the alt is text that gets displayed in case of broken requests,
such as when the image does not load, or if the user is a blind person. Note that image tags are void tags. This means that they do not have "content"
like we've seen in other kinds of tags. Since image tags have no "content", hence they also have no closing tag.
Let's say I wanted to make an image tag for a picture of the Space Needle in Seattle, my image tag would look something like this:
<img src= "https://upload.wikimedia.org/wikipedia/en/4/4f/Seattle_Space_Needle_Crop.jpg" alt = "Seattle Space Needle"> .

Whitespace

When we type several lines of text in an HTML editor, for example, and you put multiple spaces or empty lines in the middle of your text, when you render
it into HTML, the result will be one long line of text. This is what is called Whitespace. It can be avoided by using a <br> tag, which is a break tag
. Put those in your text in your HTML editor at the points where you want your sentences to break (when rendered), and the text will continue on a new line.

Inline and block elements:

Examples of inline elements are <br>, <b>, <em>, and <img> tags. The reason is that they all just have their effect in the middle of the text when rendered
into HTML. An example of a block element is the <p> tag, since it doesn't just make a break in the text to start the following text on a new line like a <br> tag would.
The <p> tag actually forms something like an invisible box that has height and width, and this box contains text in it.

Span and Div elements:

Span is an inline element, and Div is a block element. Both of them are container elements, since they contain their content. Styles can be attached to both of them
to adjust how they display their content. To show the difference, I will use both the span and div elements on identical sentences, as follows:

The first sentence shows the effect of the span element on the "Perhaps one day I will become" segment of the sentence, while the second sentence shows
the effect of the div element on the same portion of the sentence.

The composition of websites

Websites are composed of:
HTML files: which form the framework or structure of the website, like the walls of a house.
CSS files: which does the styling of the website.
Javascript files: which are the interactive components of the website, that change some of its elements.
Web tools: which are used to help build websites.

A look at webpage structure

When looking at the elements of a webpage, you don't see the actual code, but you see the structure of the page. As you move your mouse over any given element of the page, the segment of the page which each element represents become highlighted. when you click on the triangle to the left of an element, it expands, and you can continue going deeper into the element until you see the actual text that appears on the page. Note that not all elements are actually visible on the page, but all elements are rectangular.

It is notable that the structure of the page is a tree-like structure, and the technical term for this tree-like structure is DOM, which stands for Document Object Model.Each sideways triangle in this structure is an HTML element. Each one of these elements has an opening tag and a closing tag, and the contents of each element can be text or can be another element.

HTML-CSS-DOM

So, how do these three, meaning HTML, CSS, and DOM work together? Well, the web broswer turns HTML tags into elements in a tree, and it knows how to do that because of DOM. Each tag has a name or type that dfines what kind of element will be created. Also, each tag can have attributes with values. The HTML document written defines the structure and content of the page, while CSS defines the style. CSS allows you to use specific syntax and rules to determine how elements look on a page.

Design on webpages

Every element on a webpage is in a box. Even though some elements may look like they have other shapes, such as a circle, but they are still actually just boxes. The circle shape is actually just a style that is used to display the box as a circle. To see the style used to display a particular element, open up developer tools on the webpage, highlight the element, and that will display the style used for it on the style panel. If you change the style, the way the element is displayed immediately changes.

Boxifying design

When presented with a webpage design, a person should draw a box around every visual element until there's just one element in each box. Then, boxes can be transferred to an HTML document by defining each box with a <div> tag, and then putting in some place-holder content into each <div>element, to remember which box each tag refers to. Then, each tag should be marked to remember to apply style to it later, and this is done by giving a class attribute to each element. These class attributes are like labels on boxes, that remind you what's inside them. HTML tags can belong to multiple classes, so each element can be given more than one class attribute.
